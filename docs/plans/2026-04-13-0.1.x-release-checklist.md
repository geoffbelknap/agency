# `0.1.x` Release Checklist

Status: draft  
Last updated: 2026-04-13

## Goal

Ship an installable `0.1.x` release that friendly alpha testers can:

- install via Homebrew
- pull supporting images from GHCR
- complete basic setup
- create and run agents
- use the web UI without hitting obvious first-run or core workflow failures

## Release prerequisites

Before cutting a `v0.1.x` tag:

- merge [#186](https://github.com/geoffbelknap/agency/pull/186)
- confirm local branch state is clean and `main` is current
- confirm Homebrew repo access still works through:
  - `HOMEBREW_APP_ID`
  - `HOMEBREW_APP_PRIVATE_KEY`
- confirm GHCR publishing still works from Actions `GITHUB_TOKEN`
- confirm no known blocker remains in:
  - agent create/start/DM flow
  - web setup flow
  - provider setup flow

## Required release validation

### 1. CLI artifact validation

Validate before tag:

- `go build ./cmd/gateway`
- `./agency --version`
- confirm stamped fields include:
  - semver
  - commit
  - build ID

Validate after tag/release:

- GitHub release exists with expected archives
- checksums file is present
- macOS arm64 archive downloads and runs

### 2. Homebrew validation

Validate after tag/release:

- formula updated in `geoffbelknap/homebrew-tap`
- on a clean machine or shell:
  - `brew tap geoffbelknap/tap`
  - `brew install agency`
  - `agency --version`
- smoke:
  - `agency setup --help`
  - `agency doctor` or equivalent non-destructive command

### 3. GHCR image validation

Expected published images:

- `ghcr.io/geoffbelknap/agency-python-base`
- `ghcr.io/geoffbelknap/agency-comms`
- `ghcr.io/geoffbelknap/agency-knowledge`
- `ghcr.io/geoffbelknap/agency-intake`
- `ghcr.io/geoffbelknap/agency-body`
- `ghcr.io/geoffbelknap/agency-egress`
- `ghcr.io/geoffbelknap/agency-enforcer`
- `ghcr.io/geoffbelknap/agency-workspace`
- `ghcr.io/geoffbelknap/agency-web-fetch`
- `ghcr.io/geoffbelknap/agency-web`
- `ghcr.io/geoffbelknap/agency-embeddings`

Validate after tag/release:

- both `v0.1.x` and `latest` tags exist where expected
- amd64 and arm64 manifests exist
- image labels/build IDs are present and non-unknown

### 4. Product smoke test

Run on a fresh install path:

- install CLI from Homebrew
- run setup with one provider
- bring infra up
- create a `researcher` or `generalist` agent
- start the agent
- send a DM task
- confirm reply
- open web UI
- confirm:
  - Overview
  - Agents
  - Channels
  - Missions
  - Knowledge
  - Admin / Hub / Intake

### 5. Relay-hosted tester smoke

Validate if relay is part of the alpha path:

- login to relay-hosted app
- confirm app loads current build
- create or use an agent
- complete one research flow through relay-hosted UI

## Known follow-up items that should not block `0.1.x` unless they regress

- deeper Slack modal UX polish
- Drive mutation/consent polish
- relay asset publish automation
- relay/web performance tuning

Those are already tracked separately and should stay out of the critical release path unless they break tester onboarding.

## Recommended release sequence

1. Merge `#186`.
2. Re-run:
   - alpha readiness script
   - live read-only web suite
   - live mutable web suite
3. Cut release branch or release commit only if needed.
4. Tag `v0.1.x`.
5. Watch:
   - GitHub release workflow
   - image publish workflow
6. Validate Homebrew install on a clean environment.
7. Validate one clean-machine product smoke test.
8. Announce alpha install instructions to testers.

## Suggested tester install path

Assuming release artifacts succeed:

```bash
brew tap geoffbelknap/tap
brew install agency
agency setup
agency -q infra up
agency create my-researcher --preset researcher
agency -q start my-researcher
agency send my-researcher "Research and summarize: <topic>"
```

## Open questions before tag

- Should `0.1.x` ship only the CLI and local stack, or explicitly include relay in the first tester instructions?
- Do we want a single recommended provider for tester onboarding, or let testers choose among Anthropic/OpenAI/Gemini at setup time?
- Do we want to gate `0.1.x` on one clean-machine Homebrew smoke test done by hand, or require two?
