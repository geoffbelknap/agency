## Purpose

This is the operational checklist for cutting `v0.2.0-rc1` and validating the
published prerelease.

## Preconditions

Before tagging:

- all intended `0.2.x` scope changes are committed
- the worktree is clean
- the release note / announcement copy is ready enough for a prerelease
- Homebrew and GHCR publishing secrets are expected to be available in GitHub
  Actions

Current note:

- the release workflow triggers on any tag matching `v*`
- GoReleaser is configured with `prerelease: auto`, so `v0.2.0-rc1` should be
  treated as a prerelease automatically

## Local Pre-Tag Checks

Run and keep the output:

```bash
go test ./internal/openapispec ./internal/api/... ./internal/cli ./cmd/gateway
./scripts/release-readiness-check.sh preflight --version 0.2.0-rc1
./scripts/fundamentals-readiness-check.sh
./scripts/setup-wizard-readiness-check.sh
./scripts/alpha-readiness-check.sh
```

Expected result:

- all commands pass
- no new core-path regression appears

## Tagging

Create the prerelease tag from the intended release commit:

```bash
git tag v0.2.0-rc1
git push origin v0.2.0-rc1
```

## GitHub Actions To Watch

Two workflows should start from the tag:

- `.github/workflows/release.yaml`
- `.github/workflows/release-images.yml`

What to watch in `release.yaml`:

- GoReleaser completes successfully
- GitHub prerelease is created
- release archives exist for:
  - darwin amd64
  - darwin arm64
  - linux amd64
  - linux arm64
- `checksums.txt` is present
- Homebrew tap update completes

What to watch in `release-images.yml`:

- `agency-python-base` publishes first
- the remaining images publish successfully
- both `v0.2.0-rc1` and `latest` tags are produced where expected

## Published Validation

After the workflows finish:

```bash
./scripts/release-readiness-check.sh published --tag v0.2.0-rc1
```

This should validate:

- GitHub release existence
- release assets and checksums
- Homebrew formula version and asset references
- Homebrew formula checksum alignment
- GHCR image manifests for required images

## Clean-Machine Smoke

Perform at least one clean-machine or clean-shell smoke:

```bash
brew tap geoffbelknap/tap
brew install agency
agency --version
agency setup
agency -q infra up
agency create rc1-smoke --preset researcher
agency -q start rc1-smoke
agency send rc1-smoke "Reply with one short sentence confirming the released build works."
agency log rc1-smoke
agency admin doctor
```

Expected result:

- Homebrew installs the tagged prerelease build
- version output includes `0.2.0-rc1`
- infrastructure starts
- the agent reaches running
- DM path works
- logs and doctor look normal

## Release Decision After `rc1`

Promote toward `0.2.0` final if:

- published validation passes
- clean-machine Homebrew smoke passes
- GHCR pullability is confirmed
- no serious runtime or install regressions are reported

Prefer `v0.2.0-rc2` instead if:

- publish artifacts are wrong
- Homebrew formula/update is wrong
- GHCR visibility or multi-arch manifests are wrong
- first-user install/setup path fails on the released artifact

## Current Blocking Tasks Before Actual Tag Push

These are operational, not product-definition blockers:

1. commit the current `0.2.x` scope changes
2. ensure the worktree is clean
3. choose the exact release commit
4. push `v0.2.0-rc1`
5. run the published validation and clean-machine smoke
